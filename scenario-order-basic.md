# Сценарий: Создание заказа с одной пиццей

## Цель
Проверить базовый сценарий оформления заказа без замен и проблем с оплатой.

## Участники
- Клиент
- Сайт онлайн-пиццерии
- Сервис заказов (Order Service)

## Предусловия
- В каталоге есть пицца **«Маргарита»** (id = `pz_101`, доступна для заказа).
- Система доступна, основные API работают.

## Основной поток

1. Клиент открывает сайт и запрашивает список доступных пицц.
2. Клиент выбирает пиццу «Маргарита», размер *Medium*.
3. Клиент добавляет выбранную пиццу в корзину и видит обновлённую стоимость заказа.
4. Клиент переходит к оформлению заказа.
5. Клиент заполняет адрес доставки и контактные данные.
6. Клиент выбирает способ оплаты **«Наличными курьеру»**.
7. Клиент подтверждает оформление заказа.
8. Система создаёт заказ со статусом `CREATED` и отображает номер заказа клиенту.

## Постусловия
- В системе создан новый заказ со статусом `CREATED`.
- В заказе одна позиция — пицца «Маргарита» размер *Medium*.
